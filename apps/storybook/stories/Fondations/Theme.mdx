# ShelterUI Design System Theme Colors

The ShelterUI design system provides a centralized and consistent approach to managing color schemes across components and contexts.
This document explains the structure and usage of **theme colors**, emphasizing their contextual roles and adaptability for `light` and `dark` modes.

## **Theme Colors Overview**

Theme colors are organized into semantic categories to ensure clarity, reusability, and maintainability.
Each theme color is defined with precise roles, allowing developers to focus on the **purpose** of a color rather than its appearance.

The following key categories are available:

- **Primary**: The main brand color used prominently across the system.
- **Secondary**: A complementary color for accents and secondary emphasis.
- **Tertiary**: Reserved for critical elements, such as danger zones or destructive actions (e.g., delete buttons).
- **Signal**: Provides visual feedback for states like success, warning, or information.
- **Disabled**: Represents states where an element is inactive or unavailable.
- **Neutral/Background/Text**: Supports layout, typography, and accessibility.

### Light and Dark Modes

Each theme token is defined for both `light` and `dark` modes. The `light` theme is the default and is automatically applied
unless the user's system preference is set to `dark`. The `dark` theme is activated using the `prefers-color-scheme: dark` media
query or by applying the class `dark-theme` to the `<body>` element. Similarly, the `light-theme` class can be used to explicitly
enforce the light theme.

## **Structure of Theme Tokens**

Tokens are further divided into:

1. **Main Tokens** (`main`): Define primary usage for a color with different strengths (`light`, `medium`, `strong`). These variations are commonly used to create visual states like hover, focus, or active.
2. **Alt Tokens** (`alt`): Used for alternative contexts, such as backgrounds or overlays. In this case, it is the type color (e.g., text) that varies based on the token.

### **Example: Primary Color Tokens**

In the `light` theme:

- `--color-primary-main-light`: Primary color, lighter tone (used for default components).
- `--color-primary-main-medium`: Primary color, medium tone (e.g., hover effects).
- `--color-primary-main-strong`: Primary color, stronger tone (e.g., active states).
- `--color-primary-main-on`: Text color used on top of the primary color.

For alternative usage:

- `--color-primary-alt-base`: Light background variation of the primary color.
- `--color-primary-alt-onbase`: Text color for the alternative base background.
- `--color-primary-alt-onstrong`: Text color for the stronger alternative background.

## **Background Colors**

Background colors ensure proper contrast and maintain clarity in different themes:

- `--color-background-body`: The main background for body content.
- `--color-background-surface`: Surfaces like cards or modals.
- `--color-background-alt`: Secondary surface areas.
- `--color-background-overlay`: Overlays with subtle transparency.

## **Text Colors**

Text colors adapt to the theme while ensuring proper contrast and accessibility:

- `--color-text-primary`: Default text color.
- `--color-text-secondary`: Slightly muted text for less emphasis.
- `--color-text-hint`: Even softer text, often used for hints or placeholders.
- `--color-text-disabled`: Text representing disabled states.

## **Signal Colors**

Signal colors provide visual cues for system feedback:

- `--color-signal-info-background`: Background for informational alerts.
- `--color-signal-info-foreground`: Text color on informational backgrounds.
- `--color-signal-warning-background`: Warning-related backgrounds.
- `--color-signal-success-background`: Success indicators.
- `--color-signal-critical-background`: Critical or error states.

Signal colors are adjusted for both `light` and `dark` themes to ensure readability.

## **Disabled Colors**

Disabled colors represent elements that are inactive or unavailable:

- `--color-disabled-foreground`: Text or icon colors for disabled elements.
- `--color-disabled-background`: Background color for disabled elements.

## **Best Practices for Theme Tokens**

1. **Avoid direct use of primitives**: Primitive colors (e.g., `color-blue-500`) are the foundation of the design system and should only be used indirectly via tokens.
2. **Use semantic tokens**: Apply theme tokens (e.g., `primary-main-light`, `text-primary`) to ensure consistency and adapt to theme-specific contexts.
3. **Maintain accessibility**: Ensure sufficient contrast ratios between text and background colors to meet WCAG standards.

## **Variables Example in SCSS**

Theme tokens are exposed as CSS variables. Below is an example for the `primary` color tokens in the `light` theme:

```scss
:root {
  --color-primary-main-light: #{color.$color-blue-500};
  --color-primary-main-medium: #{color.$color-blue-650};
  --color-primary-main-strong: #{color.$color-blue-800};
  --color-primary-main-on: #{color.$color-neutral-00};
}
```

And in the `dark` theme:

```scss
.dark-theme {
  --color-primary-main-light: #{color.$color-blue-700};
  --color-primary-main-medium: #{color.$color-blue-650};
  --color-primary-main-strong: #{color.$color-blue-600};
  --color-primary-main-on: #{color.$color-blue-50};
}
```

## **Conclusion**

ShelterUI theme tokens offer a flexible and context-aware color system designed to adapt seamlessly to different themes and user preferences.
By leveraging semantic tokens and maintaining a strict separation between primitives and tokens, the design system ensures scalability,
consistency, and accessibility across all components.
