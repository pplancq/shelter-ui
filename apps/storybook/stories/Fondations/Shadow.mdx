# ShelterUI Design System shadow

Shadows create a perception of depth and enhance visual hierarchy by guiding the user's attention.

# Description

Shadows mimic lighting effects to give interfaces a three-dimensional appearance.
Each shadow is defined by properties such as horizontal/vertical offset, blur, extent and color.
Their use must remain subtle to ensure an elegant, accessible interface.

# Rules of use

- **Visual hierarchy**: Adjust the intensity of shadows according to the importance of the component.
  - Light shadows (0px 2px 4px rgba(0, 0, 0, 0.1)) for buttons and small interactive elements.
  - Medium shadows (0px 4px 8px rgba(0, 0, 0, 0.15)) for maps or content elements.
  - High shadows (0px 8px 16px rgba(0, 0, 0, 0.2)) for highlighted elements such as modals.
- **Theming**: In dark interfaces, use more diffuse and discreet shadows to respect aesthetics.
- **Sobriety**: Limit the use of shadows to avoid visual overload and ensure a light interface.
