# ShelterUI Design System Button

The `Button` component is a versatile UI element that supports various styles, sizes, and configurations. It can include icons, be circular, or adapt to different themes.

## Usage of the Button Component

```tsx
import { Button } from '@pplancq/shelter-ui-react';

const MyComponent = () => {
  return (
    <Button variant="default" color="primary" size="medium" onClick={() => alert('Clicked!')}>
      Click Me
    </Button>
  );
};
```

## Using the `as` Prop

The `as` prop allows you to override the default HTML element used by the `Button` component. This is useful for integrating the `Button` into other components or for semantic purposes.

For example, here is how you can use the `Button` component with the `as` prop set to `a`:

```tsx
import { Button } from '@pplancq/shelter-ui-react';

const MyComponent = () => {
  return (
    <Button as="a" href="https://example.com" target="_blank" rel="noopener noreferrer">
      Go to Example
    </Button>
  );
};
```

In this example:

- The `Button` component is rendered as an `<a>` element.
- The `href`, `target`, and `rel` attributes are passed directly to the `<a>` element.

## Usage Rules

To ensure proper consistent usage of the `Button` component, the following rules must be respected:

1. **Button without Label (Children):**

- If the `Button` does not have a `children` prop (label), it must include a `startIcon` prop.
- The `aria-label` prop must be provided to describe the purpose of the button.

<Canvas of={ButtonStories.IconOnly} sourceState="shown" />

2. **Using the `isCircle` Prop:**

- The `isCircle` prop can only be used when the `Button` does not have a `children` prop.
- This ensures that the button is visually circular and used for icon-only buttons.

Failure to follow these rules may result in accessibility issues or inconsistent UI behavior.
